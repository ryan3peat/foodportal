Create a supplier portal web application for Essential Flavours, an Australian flavour manufacturer. Use Node.js with Express backend, PostgreSQL database, and React frontend with Tailwind CSS.

Initialize the project with:
- Authentication system using JWT
- Email service integration (Nodemailer with SendGrid/SMTP)
- Environment variables for configuration
- CORS setup for API

Database schema:
1. users table:
   - id (UUID, primary key)
   - email (unique)
   - password (hashed)
   - role (enum: 'admin', 'supplier', 'procurement')
   - company_name
   - created_at
   - updated_at

2. suppliers table:
   - id (UUID, primary key)
   - supplier_name (required)
   - contact_person (required)
   - email (required, unique)
   - phone (optional)
   - location (optional)
   - moq (minimum order quantity, optional)
   - lead_times (optional)
   - payment_terms (optional)
   - certifications (JSON array - HACCP, ISO, SQF, etc.)
   - active (boolean)
   - created_by (foreign key to users)
   - created_at
   - updated_at

3. raw_materials table:
   - id (UUID, primary key)
   - material_name (required)
   - cas_number (required)
   - fema_number (optional)
   - description
   - category (enum: 'natural', 'synthetic', 'natural_identical')
   - form (enum: 'liquid', 'powder', 'paste')
   - created_at
   - updated_at

4. quote_requests table:
   - id (UUID, primary key)
   - request_number (auto-generated, format: RFQ-YYYY-XXXXX)
   - raw_material_id (foreign key)
   - quantity_needed
   - unit_of_measure
   - specifications (JSON - purity, grade, origin requirements)
   - submit_by_date (required)
   - status (enum: 'draft', 'active', 'closed', 'cancelled')
   - find_new_suppliers (boolean)
   - created_by (foreign key to users)
   - created_at
   - updated_at

5. request_suppliers table (junction):
   - request_id (foreign key)
   - supplier_id (foreign key)
   - email_sent_at
   - email_opened_at
   - response_submitted_at

6. supplier_quotes table:
   - id (UUID, primary key)
   - request_id (foreign key)
   - supplier_id (foreign key)
   - price_per_unit
   - currency (default: 'AUD')
   - moq
   - lead_time
   - validity_date
   - payment_terms
   - additional_notes
   - attachments (JSON array of file URLs)
   - submitted_at
   - status (enum: 'submitted', 'accepted', 'rejected')
```

## **MODULE 2: AUTHENTICATION & USER MANAGEMENT**
```
Implement authentication system:

1. Login page at /login:
   - Email/password authentication
   - Role-based redirection (admin → dashboard, supplier → quote submission)
   - Password reset functionality
   - Session timeout after 8 hours

2. User roles and permissions:
   ADMIN (Essential Flavours procurement team):
   - Full CRUD on suppliers
   - Create/edit/delete quote requests
   - View all quotes and analytics
   - Export data to Excel/CSV
   - Manage user accounts

   SUPPLIER:
   - View requests sent to them
   - Submit quotes
   - Update their company profile
   - View their quote history

3. Registration flow:
   - Admins can invite suppliers via email
   - Suppliers receive unique registration link
   - Auto-populate supplier info from invitation
   - Email verification required

4. Security features:
   - bcrypt for password hashing
   - Rate limiting on login attempts
   - HTTPS enforcement
   - Input sanitization
   - SQL injection prevention
```

## **MODULE 4: QUOTE REQUEST CREATION**
```
Build quote request creation interface:

1. Create Request Page (/admin/requests/new):
   
   Step 1 - Material Information:
   - Raw Material Name (required, autocomplete from database)
   - CAS Number (required, format: XXXX-XX-X)
   - FEMA Number (optional)
   - Quantity Required (number + unit dropdown)
   - Specifications:
     * Purity percentage
     * Grade (Food/Pharma/Technical)
     * Origin requirements
     * Form (Liquid/Powder/Paste)
     * Additional specifications (rich text editor)
   
   Step 2 - Request Parameters:
   - Submit Price By Date (date picker, minimum 3 business days)
   - Validity Period Required (30/60/90 days)
   - Delivery Terms (EXW/FOB/CIF/DDP)
   - Target Price (optional, hidden from suppliers)
   
   Step 3 - Supplier Selection:
   - Checkbox list of all active suppliers
   - Filter by:
     * Category capability
     * Location
     * Previous supply of material
   - "Select All" / "Clear All" buttons
   - "Find New Suppliers" checkbox (triggers external search)
   
   Step 4 - Review & Send:
   - Preview of request
   - Email preview
   - Schedule send time option
   - Send button with confirmation

2. Request Templates:
   - Save frequently requested materials as templates
   - Quick clone previous requests
